# 硅基团队：微信小游戏项目

> 6 个月目标：开发 10+ 个微信小游戏 + 社交媒体推广

## 项目概述

**终极目标（6 个月）：**
1. 开发并上线 10+ 个微信小游戏（IAA 类型，广告变现）
2. 小红书/X 平台推广，积累粉丝，赚取流量收益

**收益来源：**
- 💰 **游戏广告收入**：IAA（In-App Advertising）广告变现
- 💰 **社交媒体收益**：小红书/X 平台涨粉，通过广告、带货、知识付费变现

**核心策略：**
- 🚀 快速开发：使用游戏引擎和模板，快速复用核心代码
- 🎮 游戏复用：建立游戏模板和组件库
- 📱 平台聚焦：专注微信小游戏（降低学习成本）
- 📣 内容营销：同步进行小红书/X 平台推广

**触发条件：**

## 避免幻觉黑洞机制

### 1. 循环检测

**定义：** 同一个问题来回交流3次以上

**检测方式：**
- 每个角色维护一个问题记录
- 记录问题的提出、讨论、解决过程
- 检测到同一问题重复出现 → 触发上报

**示例：**
```
问题1：游戏无法通过微信审核
循环1：Game Dev → Game Lead → Game Dev
循环2：Game Lead → CEO → Game Lead
循环3：CEO → Boss（触发人类介入）
```

---

### 2. 时间限制

**定义：** 问题超过预期时间仍未解决

**检测方式：**
- 每个问题设定预期解决时间
- 超时自动触发上报

**示例：**
```
问题1：游戏无法通过微信审核
预期时间：1天
实际时间：2天
→ 触发上报
```

---

### 3. 无法解决标记

**定义：** 角色明确标记"无法解决"

**触发方式：**
- 角色在汇报中标记"无法解决"
- 立即触发上报

**示例：**
```
[汇报]
遇到的问题：
- 游戏无法通过微信审核，审核意见不清楚，无法修复
- [标记：无法解决] → 需要上报给Boss
```

---

### 4. 兜底机制

**人类介入的触发条件：**

| 条件 | 说明 |
|------|------|
| 循环3次 | 同一个问题循环交流3次 |
| 超时 | 问题超过预期时间仍未解决 |
| 明确标记 | 角色明确标记"无法解决" |
| 重大问题 | 影响项目整体进度的问题 |

**人类介入的方式：**
1. Boss接收上报
2. Boss决定是否介入
3. Boss介入解决问题
4. Boss将解决方案反馈给相关角色
